# 交互优化设计简报

## 1. 功能概述

一次针对整个应用的全面 UX 优化。目标是缩短操作路径深度、提升内容可读性，让工具感觉更快、更强大，但又不增加复杂度。这不是重新设计——是在已有扎实基础上的策略性打磨。

## 2. 核心用户操作

用户应该能够**粘贴链接 → 阅读摘要**，全程零多余点击；并且在**管理结果**（下载、重试、浏览历史）时，基于清晰意图的操作分组让他们永远不需要四处寻找按钮。

## 3. 设计方向

（继承自 DESIGN.md）

- **色彩策略**：克制型（Restrained），琥珀铜色强调色，暖色中性调
- **场景**：研究人员在桌前，快速粘贴链接，在处理其他任务的间隙扫读 AI 摘要
- **参考锚点**：Raycast（即时指令感）、Linear（任务聚焦的极简）、iA Writer（长文阅读舒适感）

无需逐页覆盖——克制型策略适配所有页面。

## 4. 范围

| 维度 | 设定 |
|------|------|
| **保真度** | 生产就绪级设计规格（代码级细节） |
| **广度** | 全部四个页面（转录、下载、RSS、历史） |
| **交互性** | 描述所有状态、过渡和反馈 |
| **时间意图** | 可交付品质的改进，每次一个页面 |

## 5. 优化方案

---

### 5.1 核心区域宽度：720px → 1024px

**当前问题**：`max-width: 720px` 对长文阅读偏窄，在大屏幕上中间一列显得局促，浪费屏幕空间。

**改进方案**：
- `.main` 和 `.tab-nav` 最大宽度从 `720px` 调整为 `1024px`
- 搜索结果面板同样受益——更多横向空间让 Markdown 代码块、列表等内容更从容
- 移动端断点 `@media (max-width: 560px)` 保持，不受影响

---

### 5.2 导航栏合并：Navbar + Tab-nav 统一

**当前问题**：Navbar（Logo + 主题/语言按钮）和 Tab-nav（四个页面标签）是分离的两行，占用不必要的垂直空间，视觉上割裂。

**改进方案**：
- 将 Logo、页面标签、主题/语言控件合为一行
- 布局：`[Logo AITranscriber] — [转录 | 下载 | RSS | 历史] — [🌙 | 语言 ▾]`
- 用同一行内的间距区分 Logo 区、导航区、工具区
- 移动端 Logo 和工具按钮保持在顶行，页面标签独立一行（或使用汉堡菜单收折）

---

### 5.3 结果面板操作按钮重构

**当前问题**：下载、重试、复制按钮分散在顶部操作栏和各标签页内部，重复且混乱。

**改进方案**：

1. **下载按钮放在 Tabs 同行右侧**（JS 动态定位），与 Tabs 视觉平级：
   ```
   [Transcript] [AI Summary] [Translation]          [📥 Download] [📋 Copy]
   ```
   下载和复制始终作用于当前激活的标签页，随标签页切换自动切换目标。

2. **重试按钮仅显示在「转录文本」Tab**：
   - 摘要的重试通过 `_regenerateSummaryInPlace` 原地刷新，不额外展示按钮
   - 重试按钮出现在 Transcript tab 被激活时，操作栏或 tab 同行内显示 `[🔄 Retry]`
   - Summary tab 和 Translation tab 不显示重试按钮

3. **移除标签页内部的独立按钮区域（`.tab-pane-head`）**——操作统一在 tabs 同行的右侧。

---

### 5.4 Feed-card / History-item 展开交互统一

**当前问题**：展开后标题区域随内容滚动消失，用户需要滚回顶部才能收起。两个组件的展开交互类似但实现不统一。

**改进方案**：

1. **标题区域 sticky 固定**：展开后，标题+元数据区域 `position: sticky; top: 0;` 固定在卡片顶部，背景用 `var(--surface)` + 底部阴影分隔线
2. **内容区统一 max-height**：`feed-entries` 和 `.history-body` 统一 `max-height: 420px`，超出后内部滚动
3. **隐藏滚动条**：内容区 `scrollbar-width: none`（Firefox）和 `::-webkit-scrollbar { display: none }`（Chrome/Safari），保持视觉干净
4. **点击标题区域可收起**：标题 sticky 区域点击切换展开/收起，内容区点击不收起（防止误触）
5. **手风琴模式保留**：打开一个卡片关闭其他，保持列表整洁

---

### 5.5 RSS Entries 按时间排序

**当前问题**：RSS 条目按 feed XML 原始顺序展示，未显式按发布时间排序。

**改进方案**：
- 后端 `rss_reader.py` 返回 entries 后按 `published` 字段降序排序（最新的在前）
- 前端 `_rssLoadEntries` 渲染时也做一次防御性排序（`sort((a, b) => (b.published || '').localeCompare(a.published || ''))`）

> **已确认**：后端目前未排序，前端也未排序。需两处都加。

---

### 5.6 Marked 渲染样式优化

**当前问题**：当前使用 `marked.js 9.1.2`，基础 Markdown 渲染可用，但阅读体验还有提升空间：表格、代码块、引用、列表层级和中文长文间距可以更精致。

**改进方案**：
- 保留 `marked.js`，不替换为 `markdown-it`，避免不必要的依赖切换
- 不引入代码高亮库，不增加新的 Markdown 渲染依赖
- 仅增强现有 `.md-content` CSS 样式：
  - 表格增加横向滚动、边框、表头背景和单元格间距
  - 代码块增加背景、圆角、内边距、换行/横向滚动策略
  - 行内代码使用更清晰的底色和字号
  - blockquote 增加左侧强调线、背景和更舒适的留白
  - 列表、标题、段落间距统一，优化中文长文阅读节奏

---

### 5.7 Logo 文本特效：AITranscriber Shimmer

**当前问题**：`AI<em>Transcriber</em>` 静态双色显示，缺乏辨识度。

**改进方案**：
- 为 `.nav-logo-text` 添加 shimmer 渐变动画效果：流动的光泽扫过文字
- 实现方式：
  ```css
  .nav-logo-text {
    background: linear-gradient(
      110deg,
      var(--text) 30%,
      var(--accent-text) 45%,
      var(--text) 55%
    );
    background-size: 200% 100%;
    background-clip: text;
    -webkit-background-clip: text;
    color: transparent;
    animation: logo-shimmer 3s ease-in-out infinite;
  }
  @keyframes logo-shimmer {
    0% { background-position: 200% 0; }
    100% { background-position: -200% 0; }
  }
  ```
- 动画仅在页面加载后播放一次 → 改为持续缓慢循环（3s 周期），不喧宾夺主
- `prefers-reduced-motion` 时禁用动画，降级为纯色

---

### 5.8 加载动画：自定义 CSS 微动效

**当前问题**：当前仅有 CSS spinner（转圈），视觉效果单一。

**改进方案**：
- 不引入第三方加载动画库，避免增加 CDN 依赖和样式负担
- 基于现有 CSS 组件做轻量升级：
  - 进度面板连接中状态：使用自定义多圆点追逐或脉冲动画
  - SSE 等待时的脉冲指示
  - RSS 订阅源刷新中的旋转图标保持简洁

---

### 5.9 统一管道状态（转录页面）

**当前问题**：结果面板在三个离散状态间切换（`.empty-state` → `.progress-panel.show` → `.results-panel.show`），使用 CSS `display: none/block`。切换过程感觉像是三个不同组件拼接，而非同一块区域在改变状态。

**改进方案：**

1. **即时预期状态。** 提交表单时，立即将空状态替换为极简的「正在连接…」骨架屏，不等待第一个 SSE 事件。智能进度模拟从 t=0 时刻就开始呈现。

2. **平滑状态机。** 单一容器内过渡：
   ```
   空 → 连接中（即时）→ 进度中（渐变）→ 结果（展现）
   ```

3. **持久化来源指示器。** 结果出现后，显示不可编辑的来源行：「来源：`youtube.com/watch?v=...`」，附带 ✕ 清除按钮。

**各状态：**

| 状态 | 用户所见 |
|------|----------|
| 空 | 图标 + 「粘贴链接或上传文件。支持 YouTube、Bilibili、播客等 30+ 平台。」 |
| 连接中 | 自定义 CSS 追逐/脉冲动画 + 「正在连接…」 |
| 进度中 | 当前进度面板（阶段胶囊、进度条、模式徽章） |
| 结果 | 完整结果面板，带来源指示器 |
| 错误 | 顶部错误横幅，输入框仍可使用，支持重试 |

---

### 5.10 设置——内联优先 + 状态指示器

**当前问题**：设置需要展开 → 滚动 → 填两个字段 → 点击获取 → 选择模型 → 折叠。当面板折叠时，模型状态不可见。

**改进方案：**

1. **设置按钮上的状态胶囊。** 已配置时显示绿色圆点 + 模型名，未配置时显示琥珀色圆点 + 「未配置」
2. **获取模型自动化。** API Key 和 Base URL 填写完成后 600ms 自动触发获取
3. **模型选择器放在折叠区上方。** 与语言选择器同行，API 凭证留在折叠面板内

---

### 5.11 历史 —— 可发现的批量操作

**当前问题**：「选择」按钮小而难懂，用户可能永远不会发现批量删除。

**改进方案：**

1. **选择按钮变为带计数的切换。** 非选择模式：`[选择]`，选择模式中：`[已选 (3)]`
2. **全选/取消全选** 在批量操作栏中
3. **搜索在选择模式中保持可用**
4. **来源筛选胶囊**：`[全部] [YouTube] [本地上传] [RSS]`，按来源类型筛选

8. **RSS 排序**（5.5）——entries 按时间降序
9. **Marked 渲染样式优化**（5.6）——保留 marked，仅优化 Markdown CSS 样式
10. **历史批量操作**（5.11）——筛选胶囊、选择计数

## 8. 关键状态（汇总）

| 状态 | 所在页面 | 行为 |
|-------|-------|----------|
| **首次访问** | 转录页 | 空状态，带平台提示。设置折叠，显示「未配置」圆点 |
| **已配置，等待操作** | 转录页 | 设置折叠，显示「gpt-4o」绿点。输入框已聚焦并就绪 |
| **已提交** | 转录页 | 即时自定义 CSS 加载动画 + 「正在连接…」。输入区域保持可见但禁用 |
| **处理中** | 转录页 | 进度面板，含阶段链、模式徽章、进度条 |
| **已完成** | 转录页 | 结果展现。来源指示器显示。Tabs 同行右侧下载/复制按钮 |
| **已完成 - Transcript tab** | 转录页 | 额外显示重试按钮 |
| **错误** | 任意页 | 错误横幅从顶部滑入。横幅 8 秒后自动消失 |
| **RSS 为空** | RSS 页 | 「暂无订阅。在上方粘贴 RSS 链接，或导入 JSON 文件。」 |
| **RSS 展开** | RSS 页 | 标题 sticky，内容区 max-height 420px 可滚动无滚动条 |
| **历史为空** | 历史页 | 「暂无历史。完成的转录内容将在这里显示。」 |
| **历史展开** | 历史页 | 标题 sticky，内容区 max-height 420px 可滚动无滚动条 |
| **下载中** | 下载页 | 进度面板（复用转录页面相同的进度组件） |
