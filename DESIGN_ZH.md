# 设计

## 视觉主题

**场景：** 一位研究人员坐在桌前，快速粘贴链接，在处理其他任务的间隙扫读 AI 摘要。屏幕是一个工具——被瞥一眼，而非凝视欣赏。环境光线从明亮的日光到深夜单显示器的黑暗。工具在两种光线下都必须感觉快速且不突兀。

**色彩策略：** 克制型（Restrained）。暖色中性调搭配单一琥珀铜色强调色（约占表面 10%）。暖意柔化了原本可能感觉像冰冷终端工具的印象。默认深色，因为转录文本是长文阅读，深色在 15 分钟以上的阅读中减少眼部疲劳；浅色模式为白天办公、偏好明亮界面的用户提供。

**主题：** 深色优先，浅色支持。尊重系统偏好（`prefers-color-scheme`），提供显式切换。

### 强调色策略说明

选择琥珀铜色而非蓝色（「AI 工具」的反射性选择）、绿色（「媒体应用」的反射性选择）或紫色（「开发者工具」的反射性选择）。它通过了类别反射检查：「视频转录」这个概念不会让人联想到暖琥珀色。暖意是有意为之——转录文本是长文阅读，暖色中性调营造出更接近纸张而非终端的阅读环境。

## 字体排版

### 字体栈
```
-apple-system, BlinkMacSystemFont, 'Inter', 'Segoe UI', sans-serif
```
系统字体搭配 Inter 回退。全文使用一种字体族。产品界面不需要展示字体与正文字体的配对。

### 字号层级

| 角色 | 字号 | 字重 | 行高 |
|------|------|--------|-------------|
| 页面标题 (h1) | 1.6rem – 2.2rem (clamp) | 740 | 1.15 |
| 段落标题 (h2) | 1.15rem | 650 | 1.3 |
| 正文 | 14px / 0.875rem | 400 | 1.6 – 1.8 |
| 标签/按钮文字 | 13–15px | 500–650 | 1 |
| 小号/元数据 | 10.5–12.5px | 400–500 | 1 |
| 标签（大写） | 10.5px | 650 | 1 |

层级间比例：≈1.2。足够紧凑以保持产品密度，足够对比以区分层级。

### 行长
- 散文内容（转录文本、摘要）：限制在 720px 容器宽度 → 约 65–75 字符。
- `.main` 和 `.tab-nav` 上的 `max-width: 720px` 自然实现了这一限制。

## 间距与节奏

| Token | 值 | 用途 |
|-------|-------|-------|
| `--r` | 12px | 卡片圆角 |
| `--r-sm` | 8px | 输入框/按钮圆角 |

垂直间距节奏：8px 基础单位。常用值：8, 10, 12, 14, 16, 20, 24, 36, 48, 64, 80px。并非每个间距都是 8px ——节奏来自有意图的变化，而非僵硬的倍数。

- 分区间距：36–48px
- 组件间距：12–16px
- 紧凑间距（图标+标签）：6–8px
- 输入框内边距：13px 14px（垂直对齐针对字体度量微调，而非纯倍数）

## 组件

### 按钮
- **主要按钮（`.btn-go`）**：强调色实心背景，白色文字，10px 圆角，13px 垂直内边距。用于每个页面的主要行动号召。
- **次要按钮（`.icon-btn`、`.btn-fetch`、`.btn-dl`、`.btn-copy-pane`、`.btn-upload-pill`）**：surface-2 背景，带边框，柔和文字。工具栏操作用胶囊形，内容操作用圆角矩形。
- **小按钮（`.btn-sm`）**：紧凑型，用于列表项操作。surface-3 背景，紧凑内边距。
- **开关（`.toggle-switch`）**：40×22px 胶囊，带滑动圆点。选中时显示强调色背景。
- 所有按钮均具有悬停、按下、禁用和 focus-visible 状态。

### 输入框
- **文本/URL（`.url-input`、`.s-input`）**：surface 背景，1px 边框，10px 圆角。聚焦：强调色边框 + 3px 强调色 box-shadow，12% 透明度。URL 输入框带左侧图标（链接、搜索、RSS）。
- **选择框（`.inline-lang-select`、`.s-select`）**：通过 SVG data-URI 背景图片自定义下拉箭头。选项带 surface-2 背景。
- **上传区域（`.upload-zone`）**：虚线边框，surface-2 背景。拖拽悬停：强调色边框 + 光晕。点击或拖放触发文件选择器。

### 卡片
- **订阅源卡片、历史条目**：surface 背景，1px 边框，12px 圆角。行内展开（无模态框）。展开状态：accent-dim 边框 + 微妙的 box-shadow。展开动画使用 max-height 过渡。
- **结果面板**：虚线边框，区别于可操作卡片——表示「输出区域」。

### 标签页
- **顶层导航（`.tab-nav`）**：底部边框指示器。激活状态：强调色文字 + 2px 强调色底边。
- **结果标签页（`.tab-bar`）**：相同模式，尺寸缩小，位于结果面板内。
- **下载类型标签页**：内嵌于检测结果区域。

### 进度
- **进度条**：6px 高度，surface-3 轨道，强调色填充。宽度以 500ms ease 过渡。
- **阶段胶囊（`.prog-step`）**：显示管道阶段的小圆角胶囊。状态：等待中（柔和）、当前（强调色高亮）、已完成（绿色圆点）、已跳过（变暗 + 删除线）。
- **产出物胶囊（`.artifact-pill`）**：指示输出文件就绪状态。状态：等待中（柔和）、就绪（绿色色调）。

### 空状态
- 图标 + 单行文字。居中，填充充裕。教会界面用途：「粘贴链接或上传文件。」

## 布局

### 页面结构
```
┌─ 导航栏（固定操作：主题、语言）─────┐
├─ 标签导航（转录 | 下载 | RSS | 历史）──┤
├─ 主区域（max-width: 720px，居中）────┤
│  ├─ 页面顶栏（标题 + 副标题）          │
│  ├─ 内容区                             │
│  └─ 页脚                               │
└────────────────────────────────────────┘
```

单列，居中，720px 最大宽度。无侧边栏。四个页面通过标签导航共享同一个外壳。这是有意为之的简单——这个工具不需要多列布局。

### 响应式行为
- 560px 以下：输入行纵向堆叠，设置网格变单列，历史工具栏折叠。
- 720px 容器自然适配平板和竖屏手机。
- 页面标题以下不使用流式字号。正文保持 14px。

## 动效

- 过渡：150–300ms。颜色变化 200ms，边框颜色 200ms，max-height 300ms ease-out。
- 设置展开/折叠：max-height 过渡 + 箭头旋转（250ms）。
- 按钮按下：`transform: scale(0.98)` ——即时按压反馈。
- 进度条填充：500ms ease。
- 旋转器：700ms linear 无限旋转。
- 无页面加载动画。无交错显示。无弹簧物理。

## 交互优化方案

当前设计已有扎实基础。以下针对性改进将缩短操作路径、提升可读性：

### 1. 缩短操作路径深度
- **问题**：设置需要展开 → 滚动 → 找到模型 → 获取。RSS 管理需要导航 → 添加 → 刷新 → 展开 → 点击操作。
- **方向**：持久化设置状态（已通过 localStorage 实现）。预填上次使用的模型。RSS 一键「摘要所有新增」。历史批量操作（已有选择模式——使其更容易被发现）。

### 2. 提升内容可读性
- **问题**：tab-pane-scroll 中的 Markdown 内容在 14px/1.8 行高下表现良好，但 720px 容器已接近可读行长的上限。在宽屏幕上，居中列感觉偏窄。
- **方向**：散文内容保持 720px。在 ≥900px 屏幕上将结果面板扩展至 860px。转录正文略微增大字号（14px → 15px）。

### 3. 更早呈现状态
- **问题**：进度面板仅在处理开始后隐藏空状态。用户点击「Transcribe」后在进度条出现前会看到短暂的空白。
- **方向**：在提交时立即显示骨架/预期状态，在第一个 SSE 事件到达之前就有反馈。智能进度模拟（代码中已存在 `this.sp`）——让它更快浮现。

### 4. 减少标签页摩擦
- **问题**：四个顶层标签对应四个不同的工具。只用转录功能的用户从不碰其余三个。RSS 和历史是「列表页面」，有不同的布局行为。
- **方向**：保留标签结构（已被验证且可发现）。提升列表页面布局稳定性——`list-page-active` body class 已处理此事。标签页切换无动画——内容切换应感觉像状态变化，而非页面加载。

### 5. 一键获取结果
- **问题**：转录完成后，用户看到结果标签页。要下载，点击结果面板右上角的小按钮。下载按钮与复制、重试按钮挤在一起。
- **方向**：按意图分组操作：「导出」（下载转录、摘要、翻译）和「优化」（重试、复制）。通过微妙的分隔线或间距集群进行视觉分组。

### 6. 空状态即入门引导
- **问题**：空状态写着「粘贴链接或上传文件。」但没有提示支持的平台或工作流。
- **方向**：在空状态图标下方加上一行提示，列出支持的平台（「YouTube、Bilibili、TikTok、播客等 30+ 平台」）。不是教程——只是足够传达能力的信息。
